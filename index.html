Các câu lệnh git cơ bản

  • Khởi tạo một Repository trên máy local: git init
  • Lấy code từ một Git service và download về máy và tạo Git repository local kết nối với git remote: git clone [url]
  • Kiểm tra version git: git --version
  • Kiểm tra trạng thái: git status
  • Add 1 file vào bộ nhớ tạm: git add [filename]
  • Add tất cả thay đổi vào bộ nhớ tạm: git add .
  • Lưu thay đổi trong bộ nhớ tạm thành 1 log (commit): git commit -m "[message]"
  • Kiểm tra các lịch sử commit: git log
  • Cập nhật code mới từ trên remote server và nó cũng tự động merge code từ nhánh tương ứng trên server về: git pull
  • Cập nhật thay đổi từ server nhưng không tự merge như git pull: git fetch
  • Thêm một remote repository vào local repository với tên và url: git remote add [remote_name] [url]
  • Lấy danh sách remote origin: git remote
  • Update thay đồi từ local lên server: git push origin master
  • Xóa sự thay đổi của chúng ta về trạng thái ban đầu chưa sửa hoặc về bất cứ commit nào trong quá khứ
    Reset về HEAD tức là về commit cuối cùng của nhánh đó: 
    git reset --hard HEAD       (về commit HEAD)
git reset --hard HEAD^      (về commit trước HEAD)
git reset --hard HEAD~1     (về trước 1 commit so với "^")
git reset --hard HEAD~2     (về commit trước 2 commit với HEAD)
  • Chuyển nhánh trong Git giữa nhánh nọ nhánh kia: git checkout [branch_name]
  • Config các loại trong Git repository như user, email, tool merge code, tool editor...: git config user.email "demo@gmail.com"
  • Merge code nhánh nọ vào nhánh kia: git merge nhanh_b
  • Stash từ code change hoặc apply stash: 
    git stash save --lưu change hiện tại vào stash
git stash list --hiển thị danh sách stash
git stash apply [stash_name] --đẩy changes từ 1 stash vào một nhánh
  • Chuyển nhánh or tạo nhánh mới: 
    git branch [branch_name] --tạo nhánh mới
git branch -b [branch_name] --checkout và tạo nhánh mới

Các bước hoàn thành thay đổi từ local lên github

  1. git init --> Khởi tạo một git repository trên local
  2. git add . --> add toàn bộ sự thay đổi
  3. git commit -m "[message]" --> Commit file đó vào git
  4. git branch -M master --> đổi tên nhánh thành master
  5. git remote add origin [url] --> add một remote repository tên là origin với url là repository trên Github
  6. git push -u origin master --> Push nhánh master từ local lên nhánh master trên origin

Các bước tạo nhánh và merge code

  1. git branch -b develop --> Tạo nhánh develop và thực hiện một số sự thay đổi
  2. git add .gitignore --> Tạo file gitignore để thêm các thành phần muốn bỏ qua
  3. git status --> Kiểm tra xem sự thay đổi có đúng không
  4. git add * --> Thêm file gitignore vào stage area
  5. git commit -m "Add gitignore file"
  6. git push origin develop --> đẩy nhánh develop lên remote repository
  7. git status --> Check tiếp xem có gì thay đổi và nhánh hiện tại là gì
  8. git checkout master --> Chuyển nhánh hiện tại sang master
  9. git fetch origin --> Lấy trạng thái mới nhất xem có ai commit lên master hay không
  10. git pull origin master --> Nếu có thì pull về để lấy code mới nhất
  11. git merge develop --> Merge file gitignore đang đặt ở nhánh develop sang master
  12. git push origin master --> Lại đẩy sự thay đổi ở master dưới local lên master trên server.
